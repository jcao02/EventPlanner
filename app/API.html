CasosDeUsoEventPlanner

  1. Acción que se encarga de crear un nuevo usuario e insertarlo en la base de datos.

    • URL: '/eventplanner/ACreateUser'
    • Método: POST
    • Formato de entrada y salida: JSON
    • Parámetros POST:
      • user - Requerido.
      • password - Requerido.
    • Roles: Sin restricción de rol
    • Requisitos: No disponible

    • Cursos posibles

      1. Precondición: El usuario a registrar es válido.

        Postcondicion: Se registra el usuario en la BD.

        resultado:

        • label:'/VLoginUser',
        • msg:"Usuario registrado exitosamente".

      2. Precondición: El usuario a registrar no es válido.

        Postcondicion: No hay cambio de estado.

        resultado:

        • label:'/VRegisterUser',
        • msg:"Error al crear usuario".

  2. Acción que se encarga de preparar los datos a mostrar en la vista de Registro de usuario

    • URL: '/eventplanner/VRegisterUser'
    • Método: GET
    • Formato de salida: JSON
    • Roles: Sin restricción de rol

  3. Acción que se encarga de iniciar la sesión de un usuario. La misma toma el nombre de usuario y la contraseña para verificar que estos existen y son válidos.

    • URL: '/eventplanner/ALoginUser'
    • Método: POST
    • Formato de entrada y salida: JSON
    • Parámetros POST:
      • user - Requerido.
      • password - Requerido.
    • Roles: Sin restricción de rol
    • Requisitos: No disponible

    • Cursos posibles

      1. Precondición: El usuario debe estar registrado en la BD.

        Postcondicion: Usuario exitosamente autenticado.

        resultado:

        • label:'/VHome',
        • msg:"".

        Se continúa con el rol de usuario.

      2. Precondición: El usuario no esta registrado en la BD.

        Postcondicion: True.

        resultado:

        • label:'/VLoginUser',
        • msg:"Error al iniciar sesión".

  4. Acción que se encarga de preparar los datos a mostrar en la vista de Inicio de sesión de usuario

    • URL: '/eventplanner/VLoginUser'
    • Método: GET
    • Formato de salida: JSON
    • Roles: Sin restricción de rol

  5. Acción que se encarga de listar todos los eventos en la aplicación.

    • URL: '/eventplanner/AEvents'
    • Método: GET
    • Formato de salida: JSON
      • Parámetros GET: requestedEvent
    • Roles: Sin restricción de rol
    • Requisitos: No disponible

    • Cursos posibles

      1. Precondición: True.

        Postcondicion: No hay cambio de estado.

        resultado:

        • label:'/VListEvents',
        • msg:"Se listan los eventos".

  6. Acción que se encarga de preparar los datos a mostrar en la vista de Inicio

    • URL: '/eventplanner/VHome'
    • Método: GET
    • Formato de salida: JSON
    • Roles: Sin restricción de rol

  7. Acción que se encarga de crear un nuevo evento e insertarlo en la base de datos. El usuario que esté en sesión para el momento de la creación, es el usuario dueño del evento

    • URL: '/eventplanner/ACreateEvent'
    • Método: POST
    • Formato de entrada y salida: JSON
    • Parámetros POST:
      • name - Opcional.
      • description - Requerido.
      • poster - Opcional.
      • event_date - Requerido.
      • event_location - Requerido.
      • participants - Requerido.
      • owner - Opcional.
    • Roles: usuario
    • Requisitos: No disponible

    • Cursos posibles

      1. Precondición: Evento a registrar es válido.

        Postcondicion: El evento se registra exitosamente.

        resultado:

        • label:'/VShowEvent',
        • msg:"Evento creado exitosamente".

      2. Precondición: Evento a registrar no es válido.

        Postcondicion: No hay cambio de estado.

        resultado:

        • label:'/VRegisterEvent',
        • msg:"Error al crear evento".

  8. Acción que se encarga de preparar los datos a mostrar en la vista de Listar eventos

    • URL: '/eventplanner/VListEvents'
    • Método: GET
    • Formato de salida: JSON
    • Roles: Sin restricción de rol

  9. Acción que se encarga de eliminar un evento de la base de datos. La misma recibe el ID del evento a eliminar.

    • URL: '/eventplanner/ADeleteEvent'
    • Método: GET
    • Formato de salida: JSON
    • Roles: usuario
    • Requisitos: No disponible

    • Cursos posibles

      1. Precondición: El evento debe existir, y el usuario debe ser el creador del evento.

        Postcondicion: El evento es eliminado exitosamente.

        resultado:

        • label:'/VListEvents',
        • msg:"Evento eliminado exitosamente".

      2. Precondición: El evento no existe o el usuario no es el creador del evento.

        Postcondicion: No hay cambio de estado.

        resultado:

        • label:'/VListEvents',
        • msg:"Error al eliminar evento".

  10. Acción que se encarga de preparar los datos a mostrar en la vista de Mostrar evento

    • URL: '/eventplanner/VShowEvent'
    • Método: GET
    • Formato de salida: JSON
      • Parámetros GET: eventId
    • Roles: Sin restricción de rol

  11. Acción que se encarga de preparar los datos a mostrar en la vista de nuevo evento

    • URL: '/eventplanner/VRegisterEvent'
    • Método: GET
    • Formato de salida: JSON
    • Roles: Sin restricción de rol

  12. Acción que se encarga de listar usuarios en la aplicacion. Si se usa el parametro eventid, se listan los usuarios que reservaron el evento eventid, si no se proporciona parametros, se listan todos los usuarios.

    • URL: '/eventplanner/AUsers'
    • Método: GET
    • Formato de salida: JSON
    • Roles: usuario
    • Requisitos: No disponible

    • Cursos posibles

      1. Precondición: El usuario actual es administrador o el creador del evento.

        Postcondicion: No hay cambio de estado.

        resultado:

        • label:'/VListUsers',
        • msg:"Se listan los usuarios".

  13. Acción que se encarga de preparar los datos a mostrar en la vista de listar usuarios

    • URL: '/eventplanner/VListUsers'
    • Método: GET
    • Formato de salida: JSON
      • Parámetros GET: requestedUser
    • Roles: Sin restricción de rol

  14. Acción que se encarga de generar las credenciales para asistir al evento en cuestión

    • URL: '/eventplanner/AGenerateCredentials'
    • Método: GET
    • Formato de salida: JSON
    • Roles: usuario
    • Requisitos: No disponible

    • Cursos posibles

      1. Precondición: El usuario reservó el evento.

        Postcondicion: No hay cambio de estado.

        resultado:

        • label:'/VCredential',
        • msg:"Credenciales exitosamente generadas".

      2. Precondición: El usuario no reservó el evento.

        Postcondicion: No hay cambio de estado.

        resultado:

        • label:'/VShowEvent',
        • msg:"Error al generar credenciales".

  15. Acción que se encarga de cerrar la sesión de usuario y eliminarlo de la sesión.

    • URL: '/eventplanner/ALogOutUser'
    • Método: GET
    • Formato de salida: JSON
    • Roles: usuario
    • Requisitos: No disponible

    • Cursos posibles

      1. Precondición: True.

        Postcondicion: Se elimina el usuario de la sesión.

        resultado:

        • label:'/VLoginUser',
        • msg:"Sesión exitosamente cerrada".

        Se continúa sin rol específico.

      2. Precondición: True.

        Postcondicion: No hay cambio de estado.

        resultado:

        • label:'/VHome',
        • msg:"Error al cerrar sesión".

  16. Acción que se encarga de verificar la asistencia de un usuario a un evento.

    • URL: '/eventplanner/AVerifyAssitance'
    • Método: GET
    • Formato de salida: JSON
    • Roles: admin
    • Requisitos: No disponible

    • Cursos posibles

      1. Precondición: True.

        Postcondicion: Se verifica la asistencia del usuario.

        resultado:

        • label:'/VListUsers',
        • msg:"Asistencia verificada".

      2. Precondición: True.

        Postcondicion: No hay cambio de estado.

        resultado:

        • label:'/VListUsers',
        • msg:"Error al verificar asistencia".
